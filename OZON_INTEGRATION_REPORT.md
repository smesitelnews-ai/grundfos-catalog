# Подробный отчет по интеграции Ozon Dashboard (Июль 2026)

Данный отчет описывает архитектуру, функциональные возможности и технические особенности модуля интеграции с Ozon Seller API, внедренного в проект каталога Grundfos.

## 1. Архитектура и Безопасность

* **Client-Side API Calls:** Для обхода блокировок Ozon (файрвол блокирует IP-адреса иностранных дата-центров, таких как Vercel) все запросы к Ozon Seller API выполняются исключительно на стороне клиента (в браузере) с использованием CORS-запросов через кастомную обертку `browserOzonFetch`.
* **Авторизация:** Система требует ввода `Client-Id` и `Api-Key`. Ключи безопасно сохраняются в локальное хранилище (`localStorage`) браузера пользователя и не передаются на сторонние серверы (только напрямую в API Ozon).

## 2. Структура Дашборда (Tabs)

Интерфейс продавца (`/ozon`) разделен на 4 независимые вкладки для оптимальной работы:

### 2.1. Вкладка «Сводка» (Overview)
- Содержит 4 виджета статистики, которые в точности повторяют метрики личного кабинета Ozon:
  - **Всего на Ozon** (базируется на `total` из API).
  - **В продаже** (фильтр `IN_SALE` – товары, доступные для покупки).
  - **Готовы к продаже** (фильтр `TO_SUPPLY` – товары, прошедшие валидацию, но без остатков).
  - **С ошибками** (фильтр `STATE_FAILED` / `VALIDATION_STATE_FAIL`).
- Содержит кнопку «Автоматическая синхронизация» для запуска мастера выгрузки.

### 2.2. Вкладка «Товары Ozon» (Products)
- Таблица с товарами (загружается максимум 50 товаров для мгновенного рендеринга).
- Показывает: Фото, Название, Артикул, Цену, Остатки FBO/FBS и точный статус ошибок.
- Встроен **Аккордеон характеристик**: по клику на товар отправляется запрос к `/v4/product/info/attributes` для подгрузки подробных характеристик без перегрузки основной таблицы.

### 2.3. Вкладка «Управление ценами» (Prices)
- Позволяет управлять ценообразованием (`price` и `old_price`).
- **Индикатор Индекса цен:** Опираясь на данные Ozon (`price_indexes.color_index`), система отображает, насколько цена конкурентоспособна:
  - 🟢 **Выгодная** (`COLOR_INDEX_GREEN`): ниже рынка.
  - 🟡 **Оптимальная** (`COLOR_INDEX_YELLOW`): в рынке.
  - 🔴 **Завышена** (`COLOR_INDEX_RED`): выше рынка (система также выводит минимальную цену конкурентов `minimal_price`).
- Интеграция с `/v1/product/import/prices` для мгновенного сохранения изменений.
- *Примечание:* Ozon убрал прямую передачу поля `marketing_price` (цены конкурентов) из открытого API. Индекс цен — единственный официальный способ мониторинга конкурентоспособности.

### 2.4. Вкладка «Склады FBS» (Stocks)
- Позволяет управлять остатками на собственных складах (схемы FBS/rFBS).
- Остатки Ozon (FBO) выводятся в режиме чтения (read-only).
- Интеграция с `/v2/products/stocks` для обновления остатков.

## 3. Мастер выгрузки товаров (Export Wizard)

Сложный процесс создания новых товаров разделен на 4 шага:
1. **Выбор товаров:** Чекбоксами из локальной базы Grundfos (или "Выбрать все").
2. **Финансовые настройки:** Выбор ставки НДС (0%, 10%, 20%) и установка процента наценки на базовую цену.
3. **Габариты по умолчанию:** Вес и размеры упаковки (применяются к товарам, у которых они не указаны в базе).
4. **Характеристики:** Выбор свойств для маппинга (штрихкод, габариты, расход, напор, длина и т.д.).

**Маппер атрибутов (`attributeMapper.ts`)**
При выгрузке локальные параметры конвертируются в Ozon ID. Автоматически проставляются:
- Бренд: Grundfos
- Страна-изготовитель: Германия (или иная по умолчанию)
- Гарантийный срок (в годах/месяцах)
- Специфичные тех. данные насосов.

## 4. Оптимизации и обход ограничений
- Умная пагинация и система чанков: Ozon API падает, если в массиве из 100 товаров хотя бы один содержит невалидный `product_id`. Внедрен алгоритм дробления: если чанк из 100 падает, он бьется на чанки по 10, затем по 1.
- Устранение таймаутов Vercel: За счет переноса запросов в браузер (CSR - Client-Side Rendering), удалось полностью избежать проблем Serverless-лимитов на 10-15 секунд.
